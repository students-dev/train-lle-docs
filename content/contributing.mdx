# Contributing

Help make train-lle better! We welcome contributions from the community.

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests if applicable
5. Submit a pull request

## Development Setup

```bash
# Clone the repository
git clone https://github.com/students-dev.git
cd train-lle

# Install dependencies
npm install

# Run tests
npm test

# Build the project
npm run build

# Start development server
npm run dev
```

## Code Style

- Use TypeScript for all new code
- Follow ESLint configuration
- Write meaningful commit messages
- Add JSDoc comments for public APIs

## Reporting Issues

Found a bug? Have a feature request?

1. Check existing issues
2. Create a new issue with:
   - Clear title
   - Detailed description
   - Code examples if applicable
   - Environment information

## Feature Requests

We love hearing new ideas! When requesting features:

- Explain the use case
- Describe the expected behavior
- Consider implementation complexity
- Suggest alternative approaches

## Testing

- Write unit tests for new features
- Ensure all tests pass
- Test in both Node.js and browser environments
- Include performance benchmarks for optimizations

## Documentation

- Update docs for API changes
- Add examples for new features
- Keep README up to date
- Use clear, concise language
